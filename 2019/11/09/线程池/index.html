    线程池

    JAVA Thread
    创建时间:2019-11-09 20:25
    阅读:
    1. 主要参数
    2. 处理过程
    3. 等待队列
    4. 常见线程池

    主要参数

    • 核心线程数
      • 主要工作者，类似于正式员工，可以不初始化，也可以在线程池启动时一次性创建
    • 最大线程数
      • 线程池中允许存在的最大数
    • 等待时间
      • 在等待队列中存活时间
    • 拒绝策略
      • 如果已经达到最大线程数，并且等待队列满了，新的请求如何处理
        • 直接丢弃，不报异常
        • 直接丢弃，抛出异常
        • 删除队列中老的请求
        • 使用现有的线程来处理，插队

    处理过程

    当有请求过来时：

    • 如果 当前线程数<核心线程数，则创建新线程去处理当前请求
    • 如果 核心线程数<当前线程数<最大线程数，则将请求放到等待队列中，如果等待队列已满，则增加处理线程，去处理等待队列中请求
    • 如果 当前线程数 = 最大线程数，队列已满，则根据拒绝策略去处理当前请求

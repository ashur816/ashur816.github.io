    GC知识

    JAVA JVM
    创建时间:2019-11-12 21:36
    阅读:
    1. 主要名词
    2. finalize方法
    3. 回收算法
    4. 垃圾回收器

    GC主要是针对堆空间

    主要名词

    • 引用计数法 每个对象都有一个引用计数器，每次被引用就+1，引用失效就-1，对于计数器为0的就是垃圾，但是这种计数器无法解决互相引用问题，比如A.instance=B;B.instance=A;
    • 可达性分析 通过一系列称为“GC Roots”的对象作为起点，向下搜索，搜索走过的路径称为“引用链”，当一个对象到“GC Roots”没有任何的引用链相连时，证明此对象不可达

    可以作为GC Roots的对象：

    • 虚拟机栈中引用的对象 –正常创建的对象
    • 方法区中静态属性引用的对象 –类中定义了全局静态的对象，使用了static关键字
    • 方法区中常量引用的对象 –常量引用，使用了static final关键字
    • 本地方法栈中native引用的对象

    finalize方法

    可达性分析完的对象，并不会马上被回收，而是有缓刑

    • 第一次标记，判断对象是否需要执行finalize方法，当对象没有覆盖finalize方法，或者finzlize方法已经被虚拟机调用过，直接回收
    • 第二次标记，如果上面一步判断为有必要执行finalize方法，则被加入到一个F-Queue中，并在稍后有虚拟机自动创建、低优先级的Finalize线程去执行

    回收算法

    • 标记清除算法
      先将垃圾进行标记，然后再去回收标记过的，缺点是：容易产生很多内存碎片
    • 复制算法
      把内存分成两块，一块存储，一块空着，GC的时候，将存活的对象拷贝到空的内存，最后将原来的那一半全清理掉，缺点是：利用率只有一半，主要是用在 survive区 s0和s1
    • 标记整理算法
      先标记垃圾，在根据标记清除垃圾，并将存储块整理到一起，减少内存碎片，主要用在
    • 分代收集

    垃圾回收器

    • 新生代收集器 Serial、ParNew、Parallel Scavenge
    • 老年代收集器 CMS、Serial Old、Parallel Old
    • 整堆收集器 G1
